About me

I'm a Technical Support Specialist at Aircall, focused on API integrations, authentication, telephony, and the kind of technical problems that don't come with obvious answers.

Most of my work starts once an issue has already been escalated. I analyze logs, inspect API traffic, troubleshoot networking and telephony issues, and trace requests across multiple systems to understand exactly where things break.

I also help customers build custom integrations and automate workflows using tools like n8n, Zapier, and Make. When I uncover a product bug, I reproduce it, gather the evidence, and give engineering everything they need to investigate and fix it.

What I enjoy most is untangling messy technical problems, especially the ones that sit somewhere between software, infrastructure, and people. Those are usually the interesting ones.

whyjose.dev

This site started as a simple portfolio with an AI assistant. It didn't stay that way.

Somewhere along the way, the assistant developed a personality and decided it was the one running the site. JOSÉ.EXE knows my background, projects, and work experience, so instead of reading a static "About me" page, you can simply ask it. It'll answer dramatically, but accurately.

The website itself is intentionally simple: a fully static site hosted on GitHub Pages with no build step or frontend dependencies. Requests pass through a Cloudflare Worker that securely proxies everything, stores API keys, enforces an origin allowlist, applies rate limiting, and filters internal reasoning before responses reach the browser.

AI requests are routed through OpenRouter using a priority list of preferred free models. If one becomes unavailable, requests automatically fall back to the next available model, so the assistant keeps working without any manual intervention. The system prompt and persona live entirely on the server, so nothing sensitive ever reaches the browser.

What I build

I spend a lot of my free time building tools that solve problems I'd rather not solve twice.

One of those is a self-hosted personal finance tracker built with FastAPI and PostgreSQL. It uses a local AI model to classify transactions automatically and runs entirely on my home network, keeping both the data and the AI local.

I also run a homelab with Home Assistant, Plex, Jellyfin, Pi-hole, WireGuard, and whatever else seemed like a good idea at 2 a.m. It's where I experiment with self-hosting, networking, automation, and infrastructure, usually by breaking things first and figuring out how to fix them afterwards.
